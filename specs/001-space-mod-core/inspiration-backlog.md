# Backlog заимствований для SpaceReloaded (AR + Galacticraft)

## Взять сейчас — высокая ценность, низкая цена, ложится на готовые системы

1. **Скан-отчёт ассемблера (Scan без Build)** — кнопка Scan на существующем пэде+пилоне: серверный расчёт показывает Δv по Циолковскому, TWR на старте, смещение центра масс от оси тяги и линию «Δv до орбиты 1:8 / Луны», с текстовыми причинами отказа («TWR<1», «негерметичный отсек экипажа»). Время сборки — функция объёма ракеты-сущности.
2. **Fueling Station → заправочная мачта-умбиликал** — раздельные магистрали гидролокса/керолокса от электролизёра и перегонки к пилону, конечная скорость перекачки, редстоун/компаратор «борт заправлен». Масса залитого топлива уже входит в Δv — синергия бесплатная.
3. **Единый Linker** — обобщить существующий пульт кинетической пушки в один инструмент связывания (пэд↔заправка↔ЦУП↔стыковка); связи хранить серверно по стабильным ID пэда/ракеты, а не по ссылке на сущность, чтобы переживали выгрузку чанков.
4. **Rocket Monitoring Station (ЦУП)** — консоль по обкатанному паттерну пушки: реальные кривые высоты/скорости, остаток Δv, шкала топлива, кнопка Launch, редстоун-запуск. Вся телеметрия — от честной физики, серверно.
5. **Docking Pad + Loaders/Unloaders (брать четвёркой с заправкой и ЦУП)** — именованные посадочные пэды на земле и платформе 1:8; инфраструктура линкуется к ПЭДУ и автоматически обслуживает любой севший борт; резервирование пэдов. Грузовые рукава — через трубы/хопперы и TR Energy.
6. **Полётная программа как предмет** — физический носитель в слоте бортового компьютера: цель (орбита/Луна), именованный посадочный пэд, профиль спуска. Достижимость гейтится Δv-бюджетом конкретной ракеты, а не типом чипа; копирование программ на существующих станках.
7. **Роль ракеты = следствие конструкции** — кресло→пилотируемая, сундуки→грузовая, без отдельных «типов ракет» в контенте. Возврат — только честной посадкой по программе с запасом Δv на посадочный импульс, иначе жёсткая посадка с разрушением.
8. **Звук в вакууме + Frequency Module** — вакуум не проводит звук, слышно только через радио в маске (слот-логика масок уже есть). Дёшево, а атмосферу продаёт лучше текстур.
9. **Telemetry Unit + стена экранов** — блок-экраны с данными слинкованной сущности (O2 игрока, статус полёта, герметичность модулей). Инфраструктура «блок-экран» уже есть у пушки.
10. **EVA-комплект по образцу GC** — отдельные слоты маска+обвязка+2 баллона, рендер на модели игрока, HUD-полоска O2; тиры баллонов с честной массой (входит в полезную нагрузку по Циолковскому). Заправка — только у станции-компрессора в сети TR.
11. **Обвязка герметичности** — таблица газопроницаемости блоков (шерсть/листва/гравий не держат) как datapack-тег; стоимость поддержания давления пропорциональна объёму; ручной сканер утечек, подсвечивающий путь флудфилла наружу; индикатор «герметично/нет».
12. **O2-vent вместо «пузыря»** — вентиль наддувает реальный объём через 26-флудфилл; в дырявом объёме работает, но кислород утекает в вакуум с разорительным расходом. Эффект пузыря — физически честно.
13. **UX шлюза (контроллер)** — proximity-открытие с настраиваемым радиусом, whitelist по нику, редстоун, интерлок «обе двери не одновременно»; сначала простая мембрана с потерей воздуха тамбура по флудфиллу, полный цикл откачки — позже.
14. **Балансировка топливных пар через реальные Isp/тягу** — керолокс: тяга и плотность, гидролокс: Isp; параметры пар в datapack-реестре топлив. Общий объём баков без геометрии НЕ брать — расположение баков влияет на центр масс и порядок расхода.
15. **Метеориты** — серверный спавн на телах без атмосферы, честная баллистика, кратер существующей взрывной логикой; защита — заглубление/реголитовая обваловка (толщина считается покрытием); метеоритное железо — ниша в ISRU. Конфиг частоты обязателен.
16. **Консервы** — банка+еда 2:1, возвратная тара из своей цепочки металлов; больше калорий на кг полезной нагрузки — прямая синергия с честной массой.

## Взять позже — ценно, но дорого или ждёт зависимостей

1. **Cargo-линии «пэд A → пэд B»** — полностью автоматические челночные рейсы поверхность↔платформа 1:8 поверх четвёрки из «сейчас»; вместо частот — транспондеры/маяки площадок; сервер честно считает Δv каждого рейса, нехватка = отказ запуска.
2. **Порт данных (Guidance Access Hatch)** — перезапись полётной программы стоящего/пристыкованного борта по трубе или сети данных. Нужен, только когда появятся регулярные беспилотные рейсы.
3. **Спутники как реальные объекты орбитального слоя** — сборка на существующих станках (шасси+компонент+панели, масса входит в Циолковского), выведение из бэя, серверный реестр по телам; разведспутник = целеуказание пушке, ретранслятор = условие удалённых запусков. Типы компонентов — datapack-реестр.
4. **«Данные как ресурс»** — измеримая серверная валюта от спутников/телескопа; скорость сбора от энергетики и орбиты. Фундамент для окон перелётов, целей пушки и разведки ISRU.
5. **Орбитальная съёмка (ore mapper + biome scanner в одной системе)** — спутник снимает поверхность по трассе пролёта, накапливая карту аномалий по чанкам (лёд, сланец, рельеф); точность растёт с витками; результат — предмет-карта. Вместо «живого x-ray вокруг игрока».
6. **Транспортные окна (зерно warp-механики)** — стоимость перелёта от фазового угла тел; лучший физический кусок AR, брать при выходе за пары «Земля—Луна».
7. **Открытие направлений данными** — из 26 осознанных направлений новые открываются детерминированным накоплением данных телескопа/зондов (эфемериды + окно перелёта), не лотереей; маяки — для общей видимости в мультиплеере.
8. **Обсерватория с моделью неопределённости** — оценка состава цели с погрешностью, снижаемой данными; физичные ограничения (засветка, оптика). Жемчужина AR — каркас астероидного этапа.
9. **Астероиды как реальные точки орбитального слоя** — беспилотник честно летит по Δv, таймер миссии остаётся, но исход считается из массы/Δv/оборудования, а не RNG; разведданные открывают информацию, не крутят вероятность. Лёд→гидролокс на месте, титан для лёгких баков.
10. **Wet workshop вместо «станции-в-предмете»** — выведенный борт (ракета-сущность) стыкуется к платформе и сам становится блоками-модулем; адресация именованных стыковочных портов в полётной программе, вместо уничтожения портов — герметичный переход (флудфилл готов).
11. **Физика станции вместо «ручек»** — псевдогравитация только от вращения (g = ω²r, раскрутка стоит топлива), station-keeping для высоты, ориентация влияет на панели/сектор пушки/сенсоры.
12. **Тепловая модель** — температура среды (лунный день/ночь, солнце/тень), тепловой рейтинг скафандра, климат-контроль базы на TR с расходом от объёма и ΔT по границе того же флудфилла.
13. **Solar Particle Events** — редкое событие с предупреждением за N минут, нарастающая доза без магнитосферы; защита — масса укрытия (толщина покрытия трассировкой), бонус: просадка панелей и помехи.
14. **Криокапсула** — сон в вакууме только в капсуле с питанием и O2 от сети; перемотка времени согласуется с мультиплеером и моделью света на платформе.
15. **Посадка по плотности атмосферы как принцип** — datapack-свойство тела определяет допустимые методы: парашют бесполезен в вакууме, вход в атмосферу требует теплозащиты, пропульсивная посадка жжёт Δv; Landing Balloons — беспилотная доставка без точной посадки.
16. **Марс + реактор Сабатье** — CO2-атмосфера → третье топливо (метан) на базе существующего электролизёра; жаропрочный материал с физическим смыслом (камеры сгорания) как честный гейт вместо схем из данжей.
17. **Оранжерея как второй источник O2** — фотосинтез: медленный кислород+еда за свет/энергию/CO2; «сбор из ниоткуда» выкинут.
18. **Энергоспутники + ректенна** — мощность от панелей спутников, КПД передачи, требование чистого неба (рейкаст), испепеляющий луч над ректенной оставить. После спутниковой инфраструктуры.
19. **Ровер** — электрический (TR + панели, как настоящие марсоходы), с честной массой как груз или сборкой на месте из деталей.
20. **Двухдверный шлюзовой цикл** — тамбур-объём с откачкой в сеть (медленно) или аварийным сбросом (мгновенно, воздух теряется по флудфиллу).
21. **Лунные POI без боёвки** — лавовые трубки, разбившиеся зонды, аномалии с наградой материалами/данными; ниша данжа без RPG-магии.
22. **Флаг команды (Space Race)** — блок-флаг с лицом игрока и командная статистика поверх advancements/scoreboard; дёшево, эмоционально, низкий приоритет.
23. **Газодобывающие зонды-черпаки** — бэклог этапа внешних планет: честный Δv-бюджет погружения/подъёма, водород гигантов как источник гидролокса.

## Не брать — противоречит принципам

- **Space Elevator** — мгновенная телепортация игрока; рутину late game закрывают капсула, стыковка и автоматизация грузовых рейсов.
- **Orbital Laser Drill (добыча)** — телепортация блоков с поверхности в бункер станции — запрещённая магия транспорта; ниша орбитального воздействия занята кинетической пушкой. Максимум — расчистка/остекловывание рельефа без сбора.
- **Терраформинг (biome changer + atmosphere terraformer)** — обесценивает ядро мода (герметичность, маски, ISRU) и даёт планетарный гриф в мультиплеере; фантазию закрывает паратерраформинг куполами на готовой герметичности.
- **«Станция/модуль в предмете» (Station Builder как есть)** — сворачивание построенных блоков в предмет = та же магия «ракета в инвентарь»; потребность закрыта wet workshop (см. «позже»).
- **Ручка произвольной гравитации станции** — магия; только вращение (см. «позже»).
- **Moon Dungeon + босс + ключи-схемы** — RPG-гейт прогрессии несовместим с ракетами из произвольных блоков по физике; гейтить Δv и материалами.
- **NASA Workbench с фиксированными рецептами ракет** — противоречит свободной сборке и центру масс; единственное зерно — «чертежи» как исследования, открывающие производство компонентов на станках.
- **Офф-скрин RNG-миссии по «данным»** — кубик вместо физики; заменено реальными астероидными точками (см. «позже»).
- **Общий объём топлива без геометрии баков** — убивает уже посчитанный центр масс и порядок расхода.
- **Warp Core / дилитий / FTL** — против «физики прежде всего»; спасены только transfer windows и медленный межорбитальный буксир.
- **Shield Controller (вечный щит-артефакт из данжа)** — магический предмет; вместо него расходуемые кислотостойкие покрытия, когда дойдёт до Венеры.
- **Возврат ракеты «где-то в радиусе 100 блоков»** — недетерминированная телепортация; только честная посадка по программе с ценой ошибки.
- **Unmanned Vehicle Assembler (горизонтальный гантри)** — вне текущего скоупа (нет газовых гигантов); в бэклог этапа внешних планет, не в план.
